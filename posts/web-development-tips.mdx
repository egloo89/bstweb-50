## App Router 전환 후 무엇이 달라졌나

Next.js의 App Router는 **서버 컴포넌트**를 기본으로 합니다. 데이터 페칭이 컴포넌트와 가까워지면서 보일러플레이트가 줄어들었어요.

## 1. 가능한 한 서버 컴포넌트 사용

`"use client"`는 정말 필요한 상호작용 부분에만 붙이세요. 번들 크기가 눈에 띄게 줄어듭니다.

## 2. 이미지 최적화는 `next/image`로

```tsx
import Image from "next/image"

<Image src="/hero.png" width={1200} height={600} alt="hero" priority />
```

## 3. 폰트는 `next/font`로

`GeistSans`처럼 self-host되는 폰트를 쓰면 FOIT/FOUT을 줄일 수 있습니다.

## 4. 메타데이터 API 활용

각 페이지에 `metadata`를 정의해 SEO와 OG 이미지를 깔끔하게 관리하세요.

## 5. 적절한 캐싱 전략

- 정적 데이터: 기본 캐시
- 자주 바뀌는 데이터: `revalidate` 옵션 또는 `dynamic = "force-dynamic"`

## 6. 동적 라우트의 `generateStaticParams`

빌드 시점에 정적 페이지를 생성해 응답 속도를 끌어올리세요.

## 7. Loading UI와 Suspense

`loading.tsx`로 페이지 단위 스켈레톤을 만들면 체감 성능이 좋아집니다.

## 8. Middleware로 인증과 리다이렉트

쿠키 기반 인증이나 A/B 테스트에 유용합니다.

## 9. Tailwind CSS + shadcn/ui

디자인 시스템을 빠르게 구축할 수 있고 다크 모드 지원이 쉽습니다.

## 10. 분석은 가벼운 것으로

Vercel Analytics, Plausible 같은 가벼운 도구를 추천합니다.

## 마치며

Next.js 14는 강력하지만, **기본기에 충실한 것**이 가장 빠른 길입니다. 위 10가지를 꾸준히 적용해보세요.
